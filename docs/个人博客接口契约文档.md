# 个人博客 · 接口契约文档(v1.1)

> 配套《需求与技术方案文档 v1.0》。前后端依据本文档并行开发。所有接口前缀 `/api`。

---

## 一、通用约定

### 1.1 统一返回格式

```json
{ "code": 0, "message": "ok", "data": {} }
```

- `code = 0` 成功;非 0 为业务错误,`message` 为提示文案。
- 下文各接口的「响应」均指 `data` 字段内容。

### 1.2 鉴权

- 管理接口需在请求头携带:`Authorization: Bearer <token>`。
- token 无效 / 过期 → `code` 非 0 且 HTTP `401`,前端清除 token 跳登录。

### 1.3 分页

- 请求:`page`(从 1 开始,默认 1)、`pageSize`(默认 10)。
- 响应 `PageResult<T>`:

```json
{ "list": [], "total": 100, "page": 1, "pageSize": 10 }
```

### 1.4 约定状态码(HTTP)

| HTTP | 含义 |
|---|---|
| 200 | 请求成功(业务结果看 code) |
| 400 | 参数错误 |
| 401 | 未登录 / token 失效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务端错误 |

### 1.5 状态枚举(API ↔ 数据库)

接口出入参用字符串状态(可读),数据库存 TINYINT,由后端映射:

| 含义 | API 字符串 | DB(TINYINT) |
|---|---|---|
| 文章 草稿 / 已发布 | draft / published | 0 / 1 |
| 评论 待审 / 通过 / 拒绝 | pending / approved / rejected | 0 / 1 / 2 |

| coverUrl | string/null | |

| 字段 | 类型 | 说明 |

## 三、公开接口(无需鉴权)

### 3.1 文章列表
`GET /api/articles` 
- Query:`page`、`pageSize`、`categoryId`(可选)、`tagId`(可选)
- 响应:`PageResult<ArticleListItem>`(仅已发布,按 `publishedAt` 倒序)

### 3.2 文章详情
`GET /api/articles/{id}` 
- Path:`id`
- 行为:返回详情,同时阅读量 +1(Redis 防刷:同访客时间窗内不重复计数)
- 响应:`ArticleDetail`

### 3.3 分类列表
`GET /api/categories` 
- 响应:`CategoryVO[]`(含每分类文章数)

### 3.4 标签列表
`GET /api/tags` 
- 响应:`TagVO[]`(含每标签文章数,供标签云)

### 3.5 归档
`GET /api/archives` 
- Query:`page`、`pageSize`
- 响应:`PageResult<{ id, title, publishedAt }>`(倒序,前端按年 / 月分组)

### 3.6 开源项目列表
`GET /api/projects` 
- 响应:`ProjectVO[]`(按 `sort`)

### 3.7 搜索(ES 全文检索)
`GET /api/search` 
- Query:`keyword`、`page`、`pageSize`
- 检索范围:标题 + 正文 + 标签
- 响应:`PageResult<SearchItem>`,`SearchItem` = `ArticleListItem` + `highlight`(命中高亮片段,标题 / 正文)

### 3.8 文章评论列表
`GET /api/articles/{id}/comments` 
- Path:`id`
- 响应:`CommentVO[]`(仅 approved;顶级评论嵌套其二级 `replies`)

### 3.9 提交评论
`POST /api/comments` 
- Body:

```json
{ "articleId": 1, "parentId": null, "nickname": "游客_a1b2", "email": "", "content": "写得不错" }
```

- 说明:`parentId` 为空为顶级评论,有值为挂在该顶级下的二级回复;昵称由前端自定义或随机生成;提交后状态为待审核。
- 响应:`null`(`message` 提示"评论已提交,待审核")

### 3.10 站点信息
`GET /api/site/info` 
- 响应:`SiteInfo`(供全站左栏个人信息卡与关于页)

## 四、管理接口(需 JWT)

### 4.1 管理员登录
`POST /api/admin/login` 
- Body:`{ "username": "admin", "password": "******" }`
- 响应:`{ "token": "xxx.yyy.zzz" }`

### 4.2 文章管理

#### 列表(含草稿)
`GET /api/admin/articles` 
- Query:`status`(可选:draft/published)、`keyword`(可选)、`page`、`pageSize`
- 响应:`PageResult<AdminArticleItem>`,项含 `id, title, status, categoryName, viewCount, updatedAt`

#### 详情
`GET /api/admin/articles/{id}` 
- 响应:`{ id, title, summary, content, coverUrl, categoryId, tagIds, status }`

#### 新建
`POST /api/admin/articles` 
- Body `ArticleForm`:

```json
{ "title": "标题", "summary": "摘要", "content": "# 正文 markdown",
  "coverUrl": null, "categoryId": 2, "tagIds": [1,3], "status": "published" }
```

- 行为:`status=published` 时同步写入 ES
- 响应:`{ "id": 10 }`

#### 更新
`PUT /api/admin/articles/{id}` 
- Body:`ArticleForm`
- 行为:同步更新 ES
- 响应:`null`

#### 草稿自动保存
`POST /api/admin/articles/autosave` 
- Body:`ArticleForm`(可含 `id`;无 id 新建草稿)
- 响应:`{ "id": 10 }`(供后续自动保存复用)

#### 发布 / 下架
`PATCH /api/admin/articles/{id}/status` 
- Body:`{ "status": "published" }` 或 `{ "status": "draft" }`
- 行为:发布同步写 ES,下架从 ES 移除
- 响应:`null`

#### 删除
`DELETE /api/admin/articles/{id}` 
- 行为:同步从 ES 删除
- 响应:`null`

### 4.3 图片上传(MinIO)
`POST /api/admin/upload` 
- 请求:`multipart/form-data`,字段 `file`
- 限制:类型(图片)、大小上限(如 5MB)
- 响应:`{ "url": "https://minio.example.com/blog/xxx.png" }`

### 4.4 评论审核

#### 列表
`GET /api/admin/comments` 
- Query:`status`(可选:pending/approved/rejected)、`page`、`pageSize`
- 响应:`PageResult<AdminCommentItem>`,项含 `id, nickname, content, status, articleTitle, createdAt`

#### 审核
`PATCH /api/admin/comments/{id}/status` 
- Body:`{ "status": "approved" }` 或 `{ "status": "rejected" }`
- 响应:`null`

#### 删除
`DELETE /api/admin/comments/{id}` 
- 响应:`null`

### 4.5 开源项目管理

| 操作 | 接口 |
|---|---|
| 列表 | `GET /api/admin/projects` |
| 新建 | `POST /api/admin/projects` |
| 更新 | `PUT /api/admin/projects/{id}` |
| 删除 | `DELETE /api/admin/projects/{id}` |

- Body(新建 / 更新):`{ name, description, techStack, githubUrl, sort }`
- 响应:新建返回 `{ id }`,其余 `null`

### 4.6 分类管理

| 操作 | 接口 | Body |
|---|---|---|
| 列表 | `GET /api/admin/categories` | — |
| 新建 | `POST /api/admin/categories` | `{ name }` |
| 更新 | `PUT /api/admin/categories/{id}` | `{ name }` |
| 删除 | `DELETE /api/admin/categories/{id}` | — |

### 4.7 标签管理

| 操作 | 接口 | Body |
|---|---|---|
| 列表 | `GET /api/admin/tags` | — |
| 新建 | `POST /api/admin/tags` | `{ name }` |
| 删除 | `DELETE /api/admin/tags/{id}` | — |

> 写作时也可直接传入新标签名,由后端自动创建并关联。

### 4.8 站点信息管理

| 操作 | 接口 | 说明 |
|---|---|---|
| 读取 | `GET /api/admin/site-info` | 加载当前站点信息 / 作者资料 |
| 更新 | `PUT /api/admin/site-info` | 保存昵称、简介、关于页正文、邮箱、社交链接等 |

- Body(更新):`{ nickname, avatarUrl, bio, aboutContent, email, wechat, wechatQrUrl, githubUrl, rssUrl, location, job }`
- 响应:`null`
