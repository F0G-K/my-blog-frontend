# 个人博客 · 前端开发指导文档(v1.0)

> 配套:《需求与技术方案 v1.0》《前端设计文档 v1.0》《接口契约 v1.1》。本文是前端开发者的上手指南,PC + 移动双适配。

---

## 一、目标概述

前后端分离个人博客的**前端**(单页应用 SPA),一套应用同时承载:

- **前台(读者)**:主页(文章列表)、技术分类、归档、关于作者、开源项目、文章详情(Markdown + 目录 + 评论)、搜索。
- **后台(管理员)**:登录、文章写作 / 管理、评论审核、开源项目管理、站点信息编辑。游客看不到登录 / 编辑入口。

需 **PC + 移动双适配**;支持深 / 浅色主题。

## 三、环境准备与建项目

1. 装 Node 22/24 LTS。国内可换镜像:`npm config set registry https://registry.npmmirror.com`。
2. 建项目(WebStorm 选 **Vue.js** 生成器,**取消** "Use the default project setup";或命令行):
   ```bash
   npm create vue@latest
   ```
   交互选择:**Router ✅、Pinia ✅**、ESLint ✅(建议)、Prettier(可选);TypeScript 按需(JS 起步够用)。
3. 装依赖:
   ```bash
   npm i element-plus axios markdown-it highlight.js dompurify md-editor-v3
   ```
4. 配开发代理(`vite.config` 里把 `/api` 代理到后端,避免本地跨域):
   ```js
   server: {
     proxy: { '/api': { target: 'http://localhost:8080', changeOrigin: true } }
   }
   ```

## 四、工程结构


## 五、响应式适配(双适配核心)

| 断点 | 宽度 | 布局 |
|---|---|---|
| 桌面 | ≥ 1024px | 三栏:左个人信息 + 中内容 + 右索引 |
| 平板 | 768–1023px | 两栏:中内容 + 右索引(隐藏左栏) |
| 移动 | < 768px | 单栏:仅内容;导航折叠为汉堡抽屉;个人信息 / 索引下移 |

- 顶部导航:移动端收为**汉堡按钮 → 抽屉**(含菜单、搜索、主题切换)。
- 左侧个人信息卡:桌面固定,移动进抽屉 / 底部。
- 右侧索引:列表页放最新文章 / 标签云;详情页是文章目录(TOC),移动端改为**右下角浮动按钮 → 弹出目录**。
- 主内容移动端全宽。统一管理断点变量,别散落魔法数字。

## 六、路由设计

**公开路由**:`/`(主页)、`/category`、`/tag/:name`、`/archive`、`/about`、`/projects`、`/article/:id`、`/search`。

**管理路由(需登录,路由守卫)**:`/login`、`/admin`、`/admin/articles`、`/admin/articles/edit/:id?`、`/admin/comments`、`/admin/projects`、`/admin/site-info`。

> 路由守卫:访问 `/admin/*` 检查本地 token,无效跳 `/login`。

## 七、状态管理(Pinia)

- `useThemeStore`:主题(system / light / dark)+ 切换 + 持久化。
- `useAuthStore`:token、是否登录、登录 / 登出。

## 八、与后端对接

统一前缀 `/api`,统一返回 `{ code, message, data }`(`code=0` 成功)。封装 Axios 实例:

```js
import axios from 'axios'
const request = axios.create({ baseURL: '/api', timeout: 10000 })

// 请求拦截:带 token
request.interceptors.request.use(cfg => {
  const token = localStorage.getItem('token')
  if (token) cfg.headers.Authorization = `Bearer ${token}`
  return cfg
})

// 响应拦截:解包 + 统一错误 + 401 跳登录
request.interceptors.response.use(res => {
  const { code, message, data } = res.data
  if (code === 0) return data
  ElMessage.error(message); return Promise.reject(new Error(message))
}, err => {
  if (err.response?.status === 401) { localStorage.removeItem('token'); router.push('/login') }
  return Promise.reject(err)
})
export default request
```

**状态字段是字符串**:文章 `draft/published`,评论 `pending/approved/rejected`。完整接口清单见《接口契约 v1.1》。

## 九、各页面实现要点

- **主页**:文章卡片两形态——`coverUrl` **非空→封面卡片(只标题+封面)**,空→文字卡片(标题 + 摘要 +「查看全文」)。分页。
- **文章详情**:`content` 是 Markdown 原文,前端用 markdown-it + highlight.js 渲染并生成 TOC;**渲染前用 DOMPurify 净化防 XSS**;评论区两级嵌套展示。
- **技术分类**:标签云 / 列表(带文章数),点标签进 `/tag/:name`。
- **归档**:时间线按年 / 月分组,分页。
- **关于作者**:取 `GET /api/site/info`,渲染 `aboutContent`(Markdown)+ 邮箱 / 城市 / 职位 / 公众号二维码。
- **开源项目**:卡片列表,点击跳 GitHub。
- **搜索**:调 `/api/search`,展示高亮结果;移动端搜索框全屏。

## 十、主题(深 / 浅色)

- CSS 变量定义两套色板,挂 `:root` 与 `[data-theme]`。
- 默认跟随系统(`prefers-color-scheme`),提供手动切换并持久化。
- 代码高亮主题随深 / 浅色切换。

## 十一、后台管理前端

- **登录**:账号密码 → 存 token。
- **写作页**:用 `md-editor-v3`,实时预览;**草稿自动保存**(内容变化防抖 + 定时调 `/api/admin/articles/autosave`);设封面 / 分类 / 标签 / 摘要。
- **图片上传**:编辑器上传钩子 → 调 `POST /api/admin/upload` → 拿到 MinIO 的 URL 插入 Markdown。
- **评论审核**:待审列表 → 通过 / 拒绝。
- **开源项目管理 / 站点信息编辑**:表单维护对应数据。

## 十二、建议开发顺序

1. 建项目 + Axios 封装 + 路由骨架 + 主题。
2. 通用布局(三栏 + 顶部导航)与响应式断点。
3. 主页文章列表(两种卡片)+ 分页。
4. 文章详情(Markdown 渲染 + TOC + 评论展示 / 提交)。
5. 分类 / 归档 / 关于 / 开源项目。
6. 搜索。
7. 后台:登录守卫 → 写作(编辑器 + 自动保存 + 图片上传)→ 评论审核 → 项目 / 站点信息。
8. 移动端适配收尾 + 联调。
