# 个人博客 · 需求与技术方案文档(v1.0)

> 本文档整合《需求文档 v0.4》《前端设计文档 v1.0》,并新增全栈技术选型、系统架构、接口规范、安全与性能设计,作为开发的主参考。

---

## 一、项目概述

个人技术博客,前后端分离。包含**读者前台**与**管理员后台**,复用同一前端应用;游客无需登录浏览 / 搜索 / 评论,唯一管理员经特定入口登录后可写作与管理。前端 PC + 移动双适配。

## 二、角色与权限

| 角色 | 能力 |
|---|---|
| 游客(默认) | 浏览已发布内容、搜索、提交评论(待审核) |
| 管理员(唯一) | 登录、文章写作 / 发布 / 编辑 / 删除、评论审核、开源项目管理 |

## 三、功能需求(摘要)

- **前台**:主页(文章列表,分页)、技术分类(标签)、归档(时间线,分页)、关于作者、开源项目、文章详情(Markdown + TOC + 评论)、搜索(ES 全文检索)、深 / 浅色主题。
- **评论**:留言板式,昵称可自定义或随机生成,二级楼中楼,不记录被回复对象,需审核。
- **后台**:登录、文章管理(Markdown 编辑 + 草稿自动保存)、图片上传(MinIO)、评论审核、开源项目管理。

## 四、技术选型

### 4.1 整体技术栈一览

| 层 / 组件 | 选型 | 版本                                  |
|---|---|-------------------------------------|
| 运行环境 | JDK(LTS) | **Java 21**                         |
| 后端框架 | Spring Boot | **3.5.15**                          |
| 持久层 | MyBatis-Plus | **3.5.15**(spring-boot3-starter)    |
| 鉴权 | Spring Security + jjwt | SS 随 Spring Boot 管理;jjwt **0.12.6** |
| 关系数据库 | MySQL | **8.4.9** LTS                       |
| 缓存 / 防刷 | Redis | **7.4**(官方 redis:7.4 镜像)            |
| 全文检索 | Elasticsearch | **8.18.8**(匹配 SB 3.5 客户端,勿上 9.x)    |
| 对象存储 | MinIO Java SDK | **9.0.2**                           |
| 接口文档 | Knife4j | **4.5.0**(openapi3-jakarta starter) |
| 前端框架 | Vue / Vite | Vue **3.5.38** / Vite **8.0.9**     |
| 前端运行环境 | Node(LTS) | **24.16.0 LTS**(≥ 22.12)            |
| UI 组件库 | Element Plus | **2.14.1**                          |
| 状态管理 | Pinia | **3.0.4**                           |

| 部署 | Docker / Docker Compose / Nginx | 最新稳定                                |

### 4.2 关键选型理由

- **Spring Boot 3.5.15 + Java 21**:截至 2026 年 6 月,Spring Boot 最新稳定为 4.0.x(2025-11 发布),但第三方生态(MyBatis-Plus 等)仍在跟进;3.5 线生态成熟、资料最全、国内面试最对口,是新项目的稳妥选择。3.x 起需 Java 17 及以上(本项目用 Java 21),并已从 `javax` 迁移到 `jakarta`。注:Spring Boot 3.5 的 OSS 补丁支持到 2026-06-30,个人 / 学习项目影响不大,后续可平滑升 4.0。
- **MyBatis-Plus(而非 JPA)**:贴合你已有的 MyBatis 基础与国内主流用法,CRUD / 分页 / 条件构造器开箱即用,面试常考。
- **Elasticsearch**:复用你已有的 ES 经验,做标题 + 正文 + 标签的全文检索与高亮。
- **Redis**:阅读量防刷、热点文章缓存,顺带练分布式缓存。
- **MinIO**:自建对象存储做图床,本地可 Docker 一键起,接口与 S3 兼容,后期可平滑迁云 OSS。
- **Element Plus**:组件全、文档全、响应式栅格完善,遇到问题好查。

### 4.3 依赖坑点 / 版本注意事项(重要)

- **MyBatis-Plus**:Spring Boot 3 必须用 `mybatis-plus-spring-boot3-starter`(老的 `mybatis-plus-boot-starter` 对应 SB2),当前最新 **3.5.15**。
- **JWT**:用 `jjwt 0.12.x`,其 API 与 0.11 及更早差异较大,网上多数老教程基于旧 API,别照抄。
- **MySQL**:用 **8.4 LTS**;8.0 已 EOL,9.x 为创新版(支持周期短),长期项目均不建议。
- **Elasticsearch**:用 **8.18.x**,暂不上 9.x——Spring Data Elasticsearch / 客户端兼容与生态资料尚未跟上。
- **Knife4j**:Spring Boot 3 用 `knife4j-openapi3-jakarta-spring-boot-starter`(jakarta 命名,4.5.0),用错包会因 javax / jakarta 命名空间不匹配启动报错。
- **前端**:Vite 8 要求 Node ≥ 22.12,统一锁 **Node 22 LTS**,避免版本过低构建失败。

## 五、系统架构

### 5.1 部署架构

浏览器 → **Nginx**(前端静态托管 + `/api` 反向代理)→ **Spring Boot** → MySQL / Redis / Elasticsearch / MinIO。

### 5.2 后端分层

`Controller`(接口,参数校验)→ `Service`(业务,事务,ES / Redis 协作)→ `Mapper`(MyBatis-Plus 数据访问)。统一返回对象、全局异常处理、JWT 拦截器横切于各层之上。

### 5.3 工程结构建议

后端单体即可(无需多模块):按 `controller / service / mapper / entity / dto / config / common` 分包。前端按《前端设计文档》的 `views / components / router / store / api / utils` 组织。

## 六、数据模型

| 表 | 关键字段 | 建议索引 |
|---|---|---|
| `admin` | id、username、password_hash(BCrypt) | username 唯一 |
| `article` | id、title、summary、content、cover_url、category_id、status、view_count、published_at、created_at、updated_at | category_id、status、published_at |
| `category` | id、name | name 唯一 |
| `tag` | id、name | name 唯一 |
| `article_tag` | article_id、tag_id | 联合主键(article_id, tag_id) |
| `comment` | id、article_id、parent_id、nickname、email、content、status、created_at | (article_id, status) |
| `project` | id、name、description、tech_stack、github_url、sort、created_at | sort |

`status`:文章 draft / published;评论 pending / approved / rejected。`parent_id` 为空表示顶级评论,有值表示挂在该顶级下的二级回复。

## 七、接口设计规范

- **风格**:RESTful,统一前缀 `/api`;资源用名词复数(如 `/api/articles`)。
- **统一返回**:`{ code, message, data }`;`code = 0` 成功,非 0 表示业务错误。
- **分页**:统一 `page / pageSize` 入参,返回 `{ list, total, page, pageSize }`。
- **状态码**:HTTP 语义 + 业务 code 双层;鉴权失败 401,无权限 403。
- **接口分类**:

| 类型 | 鉴权 | 示例 |
|---|---|---|
| 公开 | 否 | 文章列表 / 详情、分类、归档、开源项目列表、搜索、提交评论、已审核评论列表 |
| 管理 | JWT | 登录、文章 CRUD + 发布、草稿保存、图片上传、评论审核、开源项目 CRUD |

> 下一步可据此展开《接口契约文档》,逐接口定义 URL / Method / 入参 / 返回字段。

## 八、关键机制设计

### 8.1 鉴权(JWT)

管理员登录校验后签发 JWT;管理接口经拦截器 / Spring Security 校验 token;密码用 **BCrypt** 加密存储。单管理员账号通过初始化脚本 / 配置写入(不开放注册)。

### 8.2 Markdown 与 XSS 防护(⚠️ 重点补充)

- 后端只存 Markdown 原文,前端用 markdown-it 渲染。
- **评论内容与文章正文渲染前必须做 XSS 过滤**(如前端 DOMPurify,或后端净化),防止用户在评论 / 正文注入脚本。这是 UGC 系统的必做安全项。

### 8.3 图片上传(MinIO)

后台编辑器上传钩子 → 后端上传接口转存 MinIO → 返回可访问 URL → 插入 Markdown。需定义 Bucket、访问策略(公开读或带签名 URL)、文件类型 / 大小限制。

### 8.4 Elasticsearch 实时同步

- 索引字段:title + content + tags。
- 文章发布 / 编辑 / 删除时同步更新索引。
- **解耦建议**:个人项目可在 Service 层发布 / 删除后直接同步;若想练解耦,可用 Spring 事件(`ApplicationEvent`)异步监听,避免主流程被 ES 拖慢。

### 8.5 阅读量防刷(Redis)

同一访客(IP 或 cookie 指纹)+ 文章 ID 作为 key 写入 Redis 并设过期时间(如 30 秒),命中则不再计数,未命中才 `view_count + 1`,可异步落库减轻压力。

### 8.6 评论审核与防刷

提交即 pending,前台仅展示 approved;管理员审核通过 / 拒绝。提交接口加基本限流(如同 IP 频率限制)与内容长度校验。

### 8.7 缓存策略

文章详情 / 列表等读多写少数据可加 Redis 缓存,发布 / 编辑时失效对应缓存(Cache Aside)。

## 九、安全性设计(补充)

- 密码 BCrypt;JWT 设过期时间(可选刷新机制)。
- 所有入参做校验(`@Valid` / `Validated`)。
- SQL 注入:MyBatis-Plus 参数化查询,避免拼接。
- XSS:UGC 内容渲染前净化(见 8.2)。
- CORS:配置允许的前端来源白名单,而非全开放。
- 接口限流:评论 / 登录等敏感接口防刷。

## 十、性能与优化(补充)

- 列表 / 详情走 Redis 缓存;阅读量异步 + 防刷。
- 全文检索交给 ES,不用 MySQL `like` 模糊查。
- 图片走 MinIO(后期可接 CDN)。
- 数据库按第六节建好索引;分页避免深翻页大 offset(可用游标 / 上次 ID)。

## 十一、部署方案

- **中间件**:用 Docker Compose 一键编排 MySQL + Redis + Elasticsearch + MinIO。
- **后端**:打 jar 包,Docker 容器化运行。
- **前端**:`npm run build` 出静态文件,Nginx 托管,并反向代理 `/api` 到后端。
- 域名 + HTTPS(可选)。

## 十二、待确认 / 后续演进

1. 管理员账号初始化方式(SQL 脚本 / 启动初始化 / 配置)。
2. MinIO Bucket 命名与访问策略(公开读 vs 签名 URL)。
3. ES 同步:Service 直调 vs Spring 事件解耦,二选一。
4. **SEO**:SPA 对搜索引擎不友好;个人博客本期可接受,后续若重视收录可考虑 SSR(Nuxt)或预渲染。
5. 是否需要简单的访问统计 / 日志监控。
